# kvscf deep review тАФ 2026-07-20

Requested by Ken: the project is past MVP and in daily use; do a deep review for improvement /
DRY opportunities and a design uplift, file actionable findings as korg WIs (project `kvscf`),
then implement.

## Overall assessment

The codebase is in **good shape** тАФ much better than typical for a tool that grew feature-by-feature
across 11 sprints. Strengths worth keeping:

- **Clean layering.** `kvscf-core` (enumerate/parse/focus) is genuinely reusable; `kvscf-app` sits on
  top; the two bins are thin shells. The `remote` feature gate is done right (optional deps, CI
  asserts the no-comms binary).
- **Excellent comments.** Modules explain *why* (the AttachThreadInput recipe, the `.DEFAULT` hive
  bug, the HWND_BOTTOM two-step) with WI references. `userreg.rs` and `dock.rs` are model examples.
- **Pure parsing is pure.** `parse.rs` has no Win32 dependency and a solid test suite built from
  captured real titles.
- **Pragmatic Win32.** The focus recipe, AppBar handling, and fullscreen-yield polling are all
  minimal-but-correct with measured justifications.

The findings below are accumulation debt, not design flaws: duplication that crept in as tabs were
added (Code тЖТ Edge тЖТ Apps тЖТ favorites each cloned the row painter), a `lib.rs` that absorbed five
sprints of features, and a visual layer built ad-hoc.

## Findings

### A. DRY / structure

**A1. Four near-identical row painters** тАФ `draw_row`, `draw_fav_row`, `draw_edge_row`,
`draw_app_row` (`kvscf-app/src/lib.rs:857тАУ1161`, ~300 lines, ~80% shared). Each builds a `LayoutJob`
with the same wrap config, allocates the same row rect, paints the same hover fill, gutter marker,
and galley, and sets hover text. One parameterized row helper (gutter marker, main text, optional
never-truncated trailing host, hover text) collapses them and gives the design uplift a single
point of change. тЖТ **WI #493** (M)

**A2. App/Remote string conversions duplicated across crates** тАФ `remote.rs::app_str` ==
`winset.rs::app_key`; plus `winset.rs::app_from_key`, `remote.rs::remote_kind`. Belong in
`kvscf-core` as `App::key()` / `App::from_key()` / `Remote::kind()`. The Edge named-first sort
comparator is also written twice (`lib.rs::refresh`, `cli.rs::cmd_edge`), and `kvscf-core/lib.rs`
carries ~50 lines of repetitive per-item `#[cfg(not(windows))]` stubs that can group into one
module. тЖТ **WI #494** (S)

**A3. `split_label` reverse-parses data we had** тАФ `remote.rs:333` re-derives `(workspace, host)`
by string-splitting the `"workspace (host)"` label; a workspace named e.g. `foo (bar)` mis-splits.
`SetEntry` should carry the parts from resolve time. тЖТ **WI #495** (S)

**A4. `lib.rs` is 1,358 lines with a ~370-line `update()`** тАФ app state, chrome, three tab bodies,
Update Assist panel, four painters, eight color fns, fonts, settings, single-instance guard, all in
one file. Split into `theme` / `fonts` / `settings` / `single_instance` / `rows` modules and
per-tab render fns. Also: `run()` scans `env::args()` five separate times for probe flags тАФ one
match + a `--help` listing the probes. Minor sibling: `dock.rs` and `enumerate.rs` both hand-roll
the `GetClassNameW`/`GetWindowTextW` + UTF-16 dance. тЖТ **WI #496** (M)

### B. Robustness / perf / hygiene

**B1. Blanket `#![allow(dead_code)]` in `remote.rs` hides real dead code** тАФ `Channel.host` +
`Channel::host()` are never called (verified by grep). Remove the blanket allow, prune, use
targeted allows. тЖТ **WI #497** (XS)

**B2. `winset.rs` pure helpers untested** тАФ `parse_uri` and `percent_decode` are load-bearing
(favorites identity + relaunch) and portable, with zero tests. parse.rs and remote.rs are
well-covered; this is the gap. тЖТ **WI #498** (S)

**B3. Redundant window scans** тАФ `refresh()` runs `scan_all()`, then any new window triggers
`resolve_open_set()` which runs a *second* full `EnumWindows` + `OpenProcess` pass plus a full
workspaceStorage read; Save set and Close Extras re-scan the same way. `resolve_open_set` should
accept the already-scanned instances. Not a bug тАФ wasted work on the 1s path. тЖТ **WI #499** (S)

**B4. No `[profile.release]`** тАФ shipped exes are 6.8 / 6.2 MB with symbols. `lto`,
`codegen-units = 1`, `strip` are free wins for binaries that get copied to `C:\tools\bin` and
kwork. тЖТ **WI #500** (XS)

### C. Design uplift

**C1. No theme layer** тАФ eight ad-hoc color functions with inline RGB literals and magic spacing
numbers (pad 8.0, min row 24.0, gutter 15.0, item spacing 1.0) scattered through the painters.
`app_running_color` silently duplicates `app_color`'s Stable blue. One palette struct (dark/light)
with named tokens + spacing constants. тЖТ **WI #501** (S)

**C2. Chrome uplift** тАФ the header is the busiest part of a narrow rail: three checkboxes, a
redundant "N window(s)" line (counts already sit in the tab labels), and a refresh button. Plan:
settings behind a compact тЪЩ popup; segmented-control tab strip; spacing scale from the theme;
an **active-window indicator** (subtle accent bar on the row whose window is currently
foreground); light Update Assist panel grouping. Constraints honored: one-line rows,
host-always-kept truncation, fav-gutter alignment. тЖТ **WI #502** (M)

### Noted, deliberately not filed

- `DEFAULT_HOST = "192.168.1.144"` pinned in `remote.rs` тАФ documented homelab choice; env override
  exists.
- Token comparison is not constant-time тАФ trusted LAN, app-level gate only; out of scope.
- `eframe 0.29` is a version behind тАФ upgrade has real migration cost and zero current pain;
  revisit when a feature needs it.
- CLI `list` sorts locals-first while the app interleaves hosts тАФ both intentional per context.
- Errors from `save_favorites` are silently dropped тАФ acceptable for now; the save path also writes
  through `ua_status` where it matters.
